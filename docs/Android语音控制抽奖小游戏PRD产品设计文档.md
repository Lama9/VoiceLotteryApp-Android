# 二、产品概述

## 2.1 产品定位

一款专为幼儿园小朋友设计的语音控制抽奖小游戏，以可爱童趣的视觉风格、简单易懂的语音交互为核心，降低小朋友的操作门槛；同时提供灵活的设置功能，方便老师或家长根据实际场景（如课堂奖励、家庭互动）配置抽奖规则，兼顾趣味性与实用性。

## 2.2 核心价值

- 对小朋友：语音交互无需手动操作，降低使用难度；可爱的动画和奖品设计提升娱乐体验，增强参与感。

- 对成人（老师/家长）：支持自定义奖品、数量及概率，适配不同场景需求；操作简单，无需复杂学习即可完成配置。

## 2.3 适用场景

- 幼儿园课堂：老师用于奖励表现优秀的小朋友，提升课堂积极性。

- 家庭互动：家长与小朋友进行亲子游戏，增加互动乐趣。

- 幼儿园活动：如节日活动、毕业活动等，作为互动环节提升活动氛围。

# 三、核心功能需求

## 3.1 语音交互功能

### 3.1.1 语音唤醒与识别

- 支持核心指令识别：精准识别“开始”“停”两个关键语音指令，识别成功率不低于90%（在安静环境下）。

- 语音引导：首次使用或识别失败时，界面显示语音引导提示（如卡通人物说话气泡：“小朋友，请大声喊‘开始’来抽奖哦～”）。

- 识别反馈：识别到指令后，通过动画+音效反馈（如小喇叭跳动动画+“叮”的提示音），让小朋友明确知道指令已被接收。

### 3.1.2 指令触发逻辑

- 喊“开始”：触发抽奖开始动画，随后进入抽奖过程动画，奖品列表快速滚动。

- 喊“停”：触发抽奖停止逻辑，抽奖过程动画逐步减速直至停止，最终显示中奖结果，并播放结果动画。

- 异常处理：未识别到指令时，3秒后再次显示语音引导；连续3次识别失败，提示“小朋友，声音再大一点哦～”，同时允许成人点击界面按钮触发“开始/停”（作为备用操作）。

## 3.2 动画展示功能

### 3.2.1 抽奖开始动画

- 触发时机：识别到“开始”指令后立即触发。

- 动画内容：卡通抽奖转盘/奖池快速旋转/闪烁，搭配欢快的背景音乐；界面边缘出现彩色气泡、星星等装饰元素飘动；中间显示“抽奖开始啦！”的可爱文字。

- 时长：1-2秒，结束后无缝衔接抽奖过程动画。

### 3.2.2 抽奖过程动画

- 动画内容：奖品图标（如小贴纸、小红花、玩具模型等）在屏幕中间区域快速滚动，背景保持轻微动态（如渐变色彩流动）；背景音乐节奏加快，增强紧张感。

- 交互反馈：小朋友再次喊“停”时，动画开始逐步减速，减速过程持续0.5-1秒，避免突然停止带来的突兀感。

### 3.2.3 抽奖结果动画

- 触发时机：抽奖过程动画停止后触发。

- 动画内容：中奖奖品图标放大居中显示，周围出现烟花、彩带等庆祝动画；卡通人物（如小熊猫、小兔子）出现欢呼动作；播放喜庆的中奖音效（如“哇！中奖啦！”的童声播报）。

- 时长：3-5秒，动画结束后显示“再来一次”的引导（语音+文字）。

## 3.3 抽奖规则配置功能（设置模块）

设置模块仅对成人开放，需通过简单验证（如连续点击界面某个卡通元素3次）进入，避免小朋友误操作。

### 3.3.1 奖品级别配置

- 支持自定义奖品级别数量：默认3个级别（一等奖、二等奖、三等奖），可增减至1-5个级别，级别名称可修改（如“小能手奖”“开心奖”等）。

- 每个级别对应独立的视觉标识：不同颜色的奖牌/礼盒图标，方便小朋友区分。

### 3.3.2 奖项数量配置

- 支持为每个奖品级别设置具体奖项数量：可输入数字（1-99），或通过“+”“-”按钮调整，输入范围限制为正整数。

- 剩余数量显示：抽奖过程中，实时记录每个级别奖品的剩余数量，数量为0时该级别奖品不再参与抽奖，界面提示“该奖品已抽完啦！”。

- 重置功能：支持一键重置所有级别奖品的数量至初始配置。

### 3.3.3 中奖概率配置

- 概率设置方式：提供可视化滑块或百分比输入框，每个级别奖品的中奖概率可独立设置（0%-100%）。

- 概率校验：所有级别奖品的中奖概率之和必须为100%，否则无法保存配置，并提示“请调整概率，总和需为100%哦～”。

- 默认概率：默认一等奖10%、二等奖20%、三等奖70%。

### 3.3.4 其他设置项

- 音效开关：可开启/关闭背景音乐和音效（默认开启）。

- 动画速度调节：可调整抽奖过程动画的滚动速度（默认中等速度，适合小朋友视觉感知）。

- 语音灵敏度调节：支持高/中/低三档调节，适应不同环境噪音（默认中档）。

## 3.4 其他辅助功能

- 抽奖记录：保存最近10条抽奖结果（包含奖品级别、时间），方便老师/家长查看，支持一键清空。

- 界面锁定：进入设置模块后自动锁定抽奖界面，避免小朋友在配置过程中触发抽奖；配置完成后需手动解锁。

- 低电量提醒：当设备电量低于20%时，显示可爱的低电量动画（如小电池流泪），并提示“请及时充电哦～”。

# 四、界面设计需求

## 4.1 设计风格

整体风格：可爱、童趣、明亮，符合3-6岁小朋友的审美偏好。

- 色彩：以鲜艳柔和的马卡龙色为主（如粉色、浅蓝色、浅黄色、浅绿色），避免使用深色、冷色；色彩搭配协调，无刺眼对比。

- 元素：大量使用卡通形象（如小兔子、小熊猫、小恐龙、花朵、星星、气泡等）；图标采用圆润的圆角设计，无尖锐边缘。

- 文字：使用圆润可爱的字体（如方正少儿体、汉仪娃娃体），字号较大（最小字号不小于24号），颜色与背景对比明显，方便小朋友识别。

## 4.2 核心界面布局

### 4.2.1 首页（抽奖主界面）

- 顶部：应用名称“童趣语音抽奖小能手”+卡通logo（如旋转的小奖盘）。

- 中间：核心抽奖区域，占屏幕70%以上；初始状态显示静态奖盘/奖池背景，中间有“喊‘开始’抽奖吧！”的语音引导文字和麦克风图标。

- 底部：三个功能按钮（图标+文字），分别为“语音引导”（重新播放引导语音）、“再来一次”（备用手动触发按钮）、“设置”（进入设置模块的入口）。

### 4.2.2 设置界面

- 顶部：返回按钮（卡通箭头图标）+“设置中心”标题。

- 中间：分模块显示设置项，每个模块有独立的卡通图标（如奖品配置对应礼盒图标、概率配置对应骰子图标）；每个设置项配有简洁的文字说明，方便成人理解。

- 底部：保存按钮（卡通保存图标+“保存配置”文字）+重置按钮（“重置默认”文字）。

### 4.2.3 结果展示界面

在抽奖主界面基础上叠加结果动画，中奖奖品居中放大显示，周围环绕庆祝元素；动画结束后，底部显示“再来一次”“查看记录”两个按钮。

## 4.3 交互设计

- 触摸反馈：所有可点击元素（如按钮、设置项）被点击时，有明显的缩放动画（放大1.1倍后恢复）+颜色变化反馈。

- 引导性：首次使用时，有分步引导动画（如箭头指向麦克风图标，提示“点击这里也可以开始哦～”）。

- 容错性：设置过程中未保存退出时，弹出提示（卡通对话框：“配置未保存，确定要退出吗？”）；输入无效数字时，显示红色感叹号+“请输入正确的数字哦～”。

# 五、非功能需求

## 5.1 性能需求

- 启动速度：应用启动时间不超过3秒（在中端Android设备上）。

- 动画流畅度：所有动画帧率不低于30fps，无卡顿、掉帧现象。

- 语音识别响应：识别“开始”“停”指令的响应时间不超过1秒。

## 5.2 兼容性需求

- 系统版本：支持Android 8.0及以上版本。

- 屏幕适配：支持主流Android设备屏幕尺寸（4.7-6.7英寸），自适应布局，无界面元素遮挡、变形。

## 5.3 安全性需求

- 权限管理：仅申请必要权限（麦克风权限用于语音识别、存储权限用于保存抽奖记录），首次申请时提供简洁的权限说明（卡通对话框：“需要使用麦克风来听小朋友的指令哦～”）。

- 误操作防护：设置模块需验证后进入，避免小朋友修改配置；抽奖过程中不会触发其他系统功能（如电话、短信）。

## 5.4 稳定性需求

- 连续运行：支持连续运行1小时以上无崩溃、无内存泄漏。

- 异常恢复：遇到异常情况（如麦克风权限被拒绝、存储不足）时，能友好提示并引导用户解决，不直接崩溃。

# 六、原型交互说明（简化）

## 6.1 核心流程

打开应用→进入首页（显示语音引导）→小朋友喊“开始”→触发开始动画→进入过程动画→小朋友喊“停”→过程动画减速停止→触发结果动画→显示中奖结果→引导“再来一次”/查看记录。

## 6.2 设置流程

首页点击“设置”→完成验证（连续点击卡通元素3次）→进入设置界面→配置奖品级别/数量/概率→点击“保存配置”→返回首页（配置生效）。

# 七、备注与优先级

## 7.1 功能优先级

- P0（必须实现）：语音识别“开始/停”、三大抽奖动画、基础奖品配置（级别、数量、概率）、核心界面设计。

- P1（建议实现）：抽奖记录、语音灵敏度调节、动画速度调节。

- P2（可选实现）：自定义奖品图标、多语言支持（中文/英文）、分享抽奖结果。

## 7.2 其他备注

- 语音识别建议集成成熟的第三方SDK（如百度语音、讯飞语音），此类SDK适配中文语音场景，识别准确率高，且提供Android原生开发SDK，能更好地保障语音交互的稳定性与响应速度，提升小朋友的使用体验。

- 动画资源建议采用矢量图，保证在不同屏幕尺寸下的清晰度。

- 背景音乐和音效需选择轻柔、欢快的风格，避免音量过大刺激小朋友。
